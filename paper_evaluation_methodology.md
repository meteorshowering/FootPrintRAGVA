# 系统评估建议与可做的数据分析

本文档分为两部分：**一、基于现有实验数据可做的数据分析**（含已实现脚本与可扩展方向）；**二、系统应如何进行评估**（评估框架与建议）。

---

## 一、可做的数据分析

以下分析均基于 `experiment_results_*.json` 的会话日志结构（多轮迭代、每轮多次检索、每条证据含 retrieval score、evaluation 的 branch_action / relevance / credibility / suggested_keywords）。

### 1. 已实现的分析（见 `experiment_analysis.py`）

运行方式（在 `backandfront` 目录下）：

```bash
python experiment_analysis.py
# 或指定文件
python experiment_analysis.py mapfront/public/experiment_results_20260222_164140.json
```

脚本输出包含：

| 分析维度 | 内容 | 论文用途 |
|----------|------|----------|
| **轮次级** | 每轮检索调用次数、总返回条数、重复条数及重复率 | 展示多轮扩展与收敛：随轮次增加重复率上升（如 Round 4–6 达 44%–62%），可说明检索空间在收敛 |
| **证据级** | branch_action 分布（GROW/KEEP/PRUNE）、relevance/credibility 均值与极值 | 表格/柱状图：评估模块的决策分布与分数分布 |
| **按 action 的分数** | GROW 的 relevance/credibility 均值高于 KEEP、PRUNE 最低 | 验证「高价值证据」与高分的一致性，可画箱线图 |
| **唯一论文数** | 会话涉及的论文总数 | 说明检索覆盖的文献多样性（如单会话 24 篇） |
| **建议关键词** | 各 action 产生的 suggested_keywords 数量、TOP 关键词 | 展示 GROW 是否带来更多可扩展线索（如 GROW>KEEP>PRUNE） |
| **检索分 vs 评估分** | retrieval_score 与 relevance 的 Pearson 相关系数 | 讨论检索模型与人工/模型评估是否一致；若相关性强可说明检索排序有效，若弱可讨论评估维度不同 |

基于 20260222 会话的典型结果（可直接或改写成论文表述）：

- **轮次与收敛**：7 轮共 27 次检索调用，总返回约 260 条，重复率从 Round 0 的 0% 升至 Round 5 的 62%，符合「先扩展后收敛」的预期。
- **评估一致性**：GROW 的 relevance 均值约 9.0、credibility 约 8.3；PRUNE 约 3.5 与 6.3；KEEP 介于两者之间，说明 branch_action 与双维度评分一致。
- **检索与评估关系**：retrieval_score 与 relevance 的相关系数约 0.09（弱正相关），可解释为：检索排序主要依据语义相似度，而 relevance 还包含「对当前子意图的贡献」「是否可深挖」等，二者维度不同属合理。

导出文件 `analysis_output/experiment_analysis_export.json` 可用于绘制论文中的分布图、按轮次趋势图等。

### 2. 可扩展的分析（未在脚本中实现）

- **按轮次的 action 分布**：每轮 GROW/KEEP/PRUNE 的数量与比例，观察前期是否更多 GROW、后期是否更多 KEEP/PRUNE（收敛阶段）。
- **按策略类型**：`strategy_semantic_search` 与 `strategy_metadata_search` 分别的 GROW 率、平均 relevance，比较两种策略的效用。
- **论文级统计**：每篇论文被检索到的 chunk 数、其中 GROW 占比；识别「高产出论文」与「多被 PRUNE 的论文」，用于讨论文献库质量或检索偏差。
- **suggested_keywords 的后续命中**：若下一轮检索使用了某条证据的 suggested_keywords，该轮新证据中 GROW 比例是否更高（需对齐 round 与 query_intent）。
- **多会话对比**：对多个 `experiment_results_*.json`（不同 root_goal 或不同日期）做同一套统计，报告均值与方差，增强结论的泛化性。

## 二、系统应如何进行评估

建议从**内在评估**、**外在评估**、**对比实验**和**评估可靠性**四个维度设计评估方案；可根据论文篇幅与现有资源选取部分实施。

### 1. 内在评估（检索与评估质量）

- **检索质量**
  - **指标**：在具备标注的数据集上，计算 Precision@K、Recall@K、MRR、NDCG（若有人工标注的相关性等级）。标注可来自：专家对「根目标 + 子意图」下 chunk 的相关性/必要性打分。
  - **当前可做**：无标注时，用会话内**评估模块的 relevance 分数**作为代理标签，计算「检索返回的 Top-K 中 relevance≥7 的比例」等，与单轮检索或固定 K 的基线对比。

- **评估模块一致性**
  - **指标**：branch_action 与 relevance/credibility 的对应关系（如 GROW 是否集中在高分段）；同一 chunk 在不同轮次/不同 query_intent 下被重新评估时的一致性（若系统支持重复出现）。
  - **当前可做**：已用脚本验证「GROW 均值 > KEEP > PRUNE」，可在论文中作为评估设计合理性的证据。

- **收敛与效率**
  - **指标**：达到「用户或任务满足」所需的轮数、总检索调用数、总 token 或 API 调用成本；重复率随轮次的变化曲线。
  - **当前可做**：轮次级统计与重复率趋势已在脚本中，可直接画图并描述「多轮编排在若干轮后达到高重复率，便于设置 max_rounds 或早停策略」。

### 2. 外在评估（下游任务与用户目标）

- **任务完成度**
  - **设计**：给定根目标，由专家或用户判断「系统最终呈现的证据集合 + 过程总结」是否足以回答或支撑该问题；采用二元（是/否）或 Likert 量表（1–5）。
  - **对比**：同一批根目标下，本系统 vs. 单轮 RAG vs. 固定多轮（无编排）的「任务完成度」均值与标准差。

- **假设/结论质量**（若系统产出假设或综合结论）
  - **设计**：对系统生成的假设或摘要进行可验证性、与证据一致性、新颖性等维度打分；或与「仅用 GROW 证据」生成的摘要做对比，看编排与剪枝是否提升摘要质量。

- **用户研究**
  - **设计**：招募若干领域专家，完成「给定根目标 + 可选补充问题」的探索任务，收集：任务完成时间、补充问题次数、主观满意度（SUS 或自定义问卷）、是否成功得出「重复度/后续方向」等可操作结论。
  - **当前可做**：案例研究已模拟一条路径；真实用户研究可作为后续工作或简短试点（小样本）。

### 3. 对比实验（基线与方法消融）

- **基线**
  - **单轮检索**：同一根目标，仅做一次语义检索、取 Top-K，无编排、无评估、无多轮。
  - **固定多轮**：固定 2–3 轮、每轮用根目标或固定子问题检索，无编排器生成 query_intent、无 GROW/KEEP/PRUNE。
  - **无评估版本**：多轮编排保留，但去掉评估模块，所有检索结果均保留，比较证据集合规模与下游任务质量。

- **指标**：在相同根目标集合上，比较（1）检索/评估层面的 Precision@K、GROW 率、重复率；（2）下游层面的任务完成度、假设或摘要质量；（3）成本层面的轮数、调用数、token 数。

- **消融**
  - 仅关闭「评估模块」：观察 GROW/KEEP/PRUNE 对证据质量与下游任务的影响。
  - 仅关闭「编排器」：固定每轮用根目标检索，观察动态 query_intent 的增益。

### 4. 评估可靠性（人工与模型一致性）

- **人工标注子集**：从若干会话中抽样 50–100 条证据，由 1–2 名专家独立标注 branch_action（或 relevance/credibility），计算与系统评估的一致率、Kappa 或 Krippendorff α。
- **结论**：若一致率较高，可说明当前评估模块可作为「无人工标注」时的代理质量指标；若存在分歧，可分析分歧类型（如「边界证据」KEEP vs GROW），用于改进 prompt 或评分标准。

## 三、论文中建议的呈现方式

1. **数据分析**：用「轮次级 + 证据级 + 检索 vs 评估相关性」的统计与 1–2 张图（如按轮次重复率、按 action 的 relevance/credibility 分布），支撑「多轮收敛」「评估与高分一致」等结论。
2. **评估框架**：在「实验与评估」或「方法」中简要列出上述四类评估（内在/外在/对比/可靠性），并说明当前已做部分（如内在中的会话内统计、收敛分析），未做部分列为局限或未来工作。
3. **案例 + 数据**：案例研究展示效用，数据分析提供量化支撑，二者结合可增强说服力。

若需要，可将本评估建议精简为论文中的「5.3 评估设计」或「6. 讨论与局限」中的一小节文字表述。
