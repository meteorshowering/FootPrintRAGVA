## 2. 核心组件功能

### 2.1 RAGService 类

RAGService是系统的核心服务类，负责处理所有检索请求和多模态功能。

#### 2.1.1 初始化流程

```python
async def initialize(self, figures_file_path: Optional[str] = None):
    # 1. 设置路径
    # 2. 加载本地JSON到内存缓存
    # 3. 初始化ChromaDB向量数据库
    # 4. 初始化CLIP模型
    # 5. 初始化多模态内存数据库
    # 6. 加载向量数据（如果数据库为空）
    # 7. 重新加载多模态数据
```

#### 2.1.2 检索方法

- **语义检索**：`query_by_semantic(query_text: str, n_results: int = 5)`
  - 基于文本的语义相似性检索
  - 使用ChromaDB的默认文本嵌入
  - 返回相关度排序的结果

- **本地过滤检索**：`query_by_local_filter(keywords: List[str] = None, paper_id: str = None, figure_type: str = None)`
  - 直接在内存中进行过滤
  - 支持关键词、论文ID和图类型过滤
  - 快速、精准的过滤检索

- **多模态检索**：`query_by_multimodal(query_image_path: str = None, n_results: int = 5)`
  - 基于图片的多模态检索
  - 使用CLIP模型提取图片特征
  - 只支持图片路径输入
  - 返回基于视觉特征匹配的结果

### 2.3 智能体系统

#### 2.3.1 首席科学家 (Orchestrator)

- **职责**：规划检索策略，平衡广度和深度
- **工作流程**：
  1. 接收用户请求
  2. 生成初始检索策略
  3. 根据评估结果调整策略
  4. 决定继续或结束任务
- **策略类型**：
  - 深度挖掘 (DEEPER)：基于现有节点进一步探索
  - 广度扩展 (BROADER)：寻找新的搜索方向
两种考虑方式，上面和树状结构有关，下面和具体方法有关。
  - 多模态检索 (MULTIMODAL)：基于图片特征检索
  - 本地过滤检索 (LOCAL_FILTER)：基于内存中的数据进行过滤
  - 语义检索 (SEMANTIC)：基于文本的语义相似性检索

#### 2.3.2 工具执行者 (ToolExecutor)

- **职责**：执行检索任务
- **工作流程**：
  1. 接收检索计划
  2. 并发执行工具函数
  3. 汇总结果
  4. 返回结果给首席科学家

#### 2.3.3 评估员 (Evaluator)

- **职责**：评估检索结果
- **评估维度**：
  - 相关性
  - 可信度
  - 价值度
- **评估结果**：
  - GROW：高价值，值得进一步探索
  - KEEP：中等价值，保留作为上下文
  - PRUNE：低价值，直接剪除

#### 2.3.4 总结者 (HypothesisGenerator)

- **职责**：生成最终科学报告
- **报告结构**：
  - 核心科学假设
  - 证据链分析
  - 关键缺口与未来方向
  - 结论

### 2.3.5 迭代总结智能体 (InterationSummaryAgent)

- **职责**：对问答过程进行系统性总结与深度分析，生成词云数据，评估检索质量
- **核心功能**：
  1. **过程分析与总结**：对问答过程中产生的问题及搜索到的内容进行系统性总结与深度分析
  2. **内容可视化**：为问题的所有结果实现词云生成功能，直观展示关键主题与高频词汇
  3. **检索质量评估**：评估问题提出的积极意义和价值，分析其对知识获取的贡献度

- **工作流程**：
  1. 接收Orchestrator发送的总结请求，包含完整的实验结果数据
  2. 分析实验结果中的迭代数据，生成简洁的过程总结
  3. 统计关键词出现频率，生成词云所需的词频数据
  4. 基于历史检索计划和单轮结果评估数据进行综合分析，评估检索质量
  5. 保存总结结果到实验结果中，并通过WebSocket返回给前端

- **评估维度**：
  - 相关性：检索结果与问题的相关程度
  - 准确性：检索结果的准确程度
  - 权威性：检索结果的权威程度
  - 完整性：检索结果的完整程度

- **与其他智能体的交互**：
  - 接收Orchestrator发送的SummaryRequest消息
  - 发送SummaryResponse消息给Orchestrator
  - 通过WebSocket将总结结果实时返回给前端

## 4. 技术栈

| 类别 | 技术 | 用途 |
|------|------|------|
| 编程语言 | Python | 后端开发 |
| 框架 | FastAPI |  API服务 |
| 向量数据库 | ChromaDB | 存储和检索向量 |
| 多模态模型 | CLIP (openai/clip-vit-base-patch32) | 图片特征提取 |

| 异步支持 | asyncio | 并发执行任务 |

## 8. 总结

ToRAGLENS后端实现了一个完整的多模态检索系统，结合了向量数据库、CLIP模型和多智能体系统，能够处理复杂的科学文献调研任务。系统具有良好的模块化设计，支持多种检索方式，特别是基于图片的多模态检索，为用户提供了强大的科研工具。

通过智能体系统的协作，ToRAGLENS能够自动规划检索策略、执行检索任务、评估结果并生成最终报告，实现了科研文献调研的自动化和智能化。
