# ToRAGLENS 后端实现文档 - 论文材料（文字版）


## 一、系统概述

### 1.1 系统定位
ToRAGLENS 是一个基于多智能体协作的科学知识发现系统，通过多轮迭代的文献检索、评估和推理，构建科研证据树，最终生成高质量的科学回答报告。

### 1.2 核心目标
系统旨在实现深度探索与广度扩展的平衡，通过动态检索策略构建结构化的证据链，对检索结果进行智能评估筛选，并综合所有证据生成结构化的科学报告。

### 1.3 技术架构
系统采用多智能体协作架构，基于 AutoGen Core 框架实现。整体架构分为四层：前端接口层（FastAPI + WebSocket）、智能体系统层（5个专业智能体）、工具执行层（多种检索工具）、数据存储层（ChromaDB 向量数据库 + 本地 JSON 缓存）。

---

## 二、核心智能体系统

### 2.1 OrchestratorAgent（首席科学家/协调者）

**职责定位**：OrchestratorAgent 是整个系统的核心控制单元，负责接收用户查询、初始化科研证据树、基于评估反馈动态规划检索策略、平衡广度探索和深度挖掘，并控制整个工作流的执行流程。

**核心数据结构**：系统维护一个科研证据树（ResearchGraph），包含根节点和多个证据节点。每个节点包含唯一标识、类型（根节点或证据节点）、状态（活跃/待处理/已剪枝）、内容信息（标题、摘要、洞察）、元数据（论文ID、关键词等）、父子关系、创建轮次、搜索历史、来源工具和参数、评估结果等完整信息。

**决策策略**：系统采用两种核心策略。深度挖掘策略（DEEPER）在评估人员报告高价值节点时触发，基于节点的论文ID或关键词进行精确检索，旨在将孤立线索扩展为完整证据链。广度扩展策略（BROADER）在检索结果重复率高或结果过少时触发，基于新关键词进行语义检索，旨在扩展搜索范围发现新的研究方向。每个策略生成前需通过相关性评分和新颖性评分双重验证（均需≥7分），每轮可生成1-5个并发策略。

**工作流程**：系统首先使用用户问题本身进行初始搜索，然后进入迭代循环。每轮迭代包括：工具执行者并发执行检索任务、协调者处理检索结果并更新图谱、评估者分批评估新证据、协调者根据评估结果规划下一轮策略。系统通过批处理计数器和轮次计数器控制流程，支持最多7轮迭代。

**实验数据记录**：系统完整记录每轮迭代的原始数据，包括用户原始问题、每轮检索策略、检索结果、评估结果等，形成完整的实验数据链，支持后续分析和可视化。

### 2.2 ToolExecutorAgent（工具执行者）

**职责定位**：ToolExecutorAgent 负责接收协调者的批量检索任务，并发执行多个检索工具，汇总结果并返回给协调者。

**实现机制**：系统采用异步并发机制，将多个检索任务同时执行，通过异步等待机制汇总所有结果后统一返回。每个工具执行结果包含原始策略信息、检索到的证据列表和可能的错误信息。

**支持的检索工具**：系统提供五种检索工具。语义检索工具基于向量相似度进行语义匹配，支持自然语言查询。元数据检索工具基于论文ID、关键词、图表类型进行精确匹配，支持快速过滤。多模态检索工具基于图片特征进行相似度检索，使用CLIP模型提取图片特征。向量检索工具直接基于输入向量进行相似度检索。证据相似度检索工具基于现有证据节点的嵌入向量查找相似证据。

### 2.3 EvaluatorAgent（评估者）

**职责定位**：EvaluatorAgent 负责对检索到的每条新证据进行单点评估，判断证据价值，提取关键洞察和推荐关键词，为协调者提供决策依据。

**评估标准**：系统采用三级评估标准。高价值证据（GROW）与问题强相关且包含进一步深挖的线索，中等价值证据（KEEP）与问题相关但可能只是背景介绍，低价值证据（PRUNE）与问题无关或信息质量低。评估维度包括相关性、可信度、准确性、权威性、完整性五个方面，每个维度评分0-10分。评估输出包括提取的关键洞察、推荐的专业名词或关键词、评估理由等。

**实现机制**：系统将新证据分批发送给评估者（每批5条），避免单次评估过多证据导致大语言模型上下文过长。评估者构建包含证据完整信息的评估提示，调用大语言模型进行评估，解析评估结果并返回评估报告。系统通过批处理计数器确保所有评估完成后再进入下一阶段。

### 2.4 InteractionSummaryAgent（交互总结智能体）

**职责定位**：InteractionSummaryAgent 在每轮迭代结束后对当前轮次进行综合总结，生成词云数据展示关键主题，评估检索质量分析优势和不足，为前端可视化提供数据支持。

**核心功能**：过程总结生成功能基于用户原始输入、当前轮次检索策略、检索结果统计、评估分析等信息，调用大语言模型生成自然语言总结。词云数据生成功能统计查询参数中的关键词、评估中的建议关键词、元数据中的关键词、当前轮次证据数据中的关键词，并赋予不同权重，生成词频统计和关键词列表。检索质量评估功能计算相关性、准确性、权威性、完整性四个维度的平均评分，分析问题的积极意义和价值，根据评分判断不足并生成改进建议。

**数据推送**：系统通过 WebSocket 实时推送总结结果到前端，支持前端实时展示词云、质量评估等信息。

### 2.5 HypothesisGeneratorAgent（假设生成者）

**职责定位**：HypothesisGeneratorAgent 在所有检索和评估完成后，综合所有活跃节点的证据，生成最终的科学回答报告。

**实现机制**：系统获取所有活跃节点（ACTIVE状态）的摘要信息，构建包含高质量证据链的报告生成提示，调用大语言模型生成结构化的科学报告。报告结构包括核心科学假设、证据链分析、关键缺口与未来方向、结论四个部分。系统通过 WebSocket 推送最终报告到前端。

## 三、RAG检索服务

### 3.1 数据存储架构

系统采用双层数据存储架构。向量数据库层使用 ChromaDB 进行持久化存储，支持向量相似度检索。本地缓存层使用 JSON 文件进行快速过滤查询，支持基于规则的精确匹配。系统还支持多模态检索，使用 CLIP 模型进行图片特征提取和相似度匹配。

### 3.2 集合管理

系统支持多个向量集合，包括文本加图片混合集合、纯文本集合、多模态集合等。集合管理器负责集合的创建、删除、获取等操作，支持动态切换不同集合以满足不同检索需求。

### 3.3 核心检索接口

语义检索接口基于向量相似度进行语义匹配，支持自适应阈值调整机制。当初始检索结果不足时，系统会自动降低分数阈值并重试，确保返回足够数量的结果。本地过滤检索接口直接在内存中的 JSON 列表进行 Python 过滤，支持论文ID精确匹配、图表类型模糊匹配、关键词交集匹配，返回置信度为100%的规则匹配结果。多模态检索接口使用 CLIP 模型提取查询图片特征，在多模态集合中进行向量检索，返回相似图片结果。向量检索接口直接基于输入向量进行相似度检索。证据相似度检索接口获取指定证据的嵌入向量，使用该向量进行相似度检索并过滤掉自身。

### 3.4 结果格式化

所有检索接口统一返回格式，包含证据ID、结构化内容（标题、摘要、洞察）、相似度分数、完整元数据、类型标识、来源文件等信息，确保后续处理的一致性。

## 四、WebSocket实时通信

### 4.1 连接管理

系统通过 ConnectionManager 管理所有 WebSocket 连接，维护活跃连接列表。当新连接建立时，系统接受连接并加入活跃列表。当连接断开时，系统自动从列表中移除。系统支持多个客户端同时连接，实现实时数据推送。

### 4.2 数据推送机制

系统支持三种类型的数据推送。图谱推送将科研证据树序列化为 JSON 格式，广播到所有活跃连接，实现图谱状态的实时更新。总结推送支持两种类型：结构化的总结响应对象和字符串类型的最终报告，通过统一的 JSON 格式推送到前端。普通 JSON 推送支持任意 JSON 数据的广播，系统会遍历所有活跃连接逐个发送，并处理发送失败的情况，自动移除失效连接。

### 4.3 消息路由

前端到后端的消息包括启动查询请求和卡片扩展检索请求。后端到前端的消息包括图谱对象更新和总结报告推送。系统通过消息类型字段进行路由分发，确保消息正确传递到对应的处理逻辑。

## 五、数据模型与协议

### 5.1 核心数据模型

证据相关模型包括原始证据项和评估结果。原始证据项包含唯一标识、来源工具、内容信息、元数据、相似度分数、工具参数等。评估结果包含目标证据ID、分支动作、提取的洞察、各项评分、评估理由、推荐关键词等。

图谱相关模型包括图谱节点和科研证据树。图谱节点包含节点标识、类型、状态、内容、元数据、父子关系、命中次数、创建轮次、搜索历史、来源信息、评估结果等完整信息。科研证据树包含根目标、所有节点、操作历史、证据唯一标识集合等。

### 5.2 消息协议

系统定义了完整的消息协议体系。用户请求包含查询内容。协调者计划包含动作类型、工具名称、父节点、参数、原因、执行结果统计等。工具输出消息包含原始计划和检索结果列表。评估请求包含新证据列表。评估报告消息包含评估结果列表和全局建议。总结请求包含实验结果、当前证据数据、当前问题、用户原始输入等。总结响应包含过程总结、词云数据、质量评估、时间戳等。

### 5.3 实验数据模型

系统定义了完整的实验数据模型，包括检索结果、查询结果、迭代结果、实验结果等层次结构。每个层次都包含完整的原始数据和统计信息，支持后续分析和可视化。

## 六、工作流执行流程

### 6.1 完整执行流程

系统执行流程从用户提交查询开始。协调者接收用户请求后初始化科研证据树并创建根节点，然后使用用户问题本身执行初始搜索。工具执行者并发执行检索工具，返回检索结果列表。协调者处理检索结果，进行去重检测，创建或更新节点，构建图谱结构。系统将新证据分批发送给评估者，每批5条。评估者对每条证据进行评估，返回评估报告。协调者更新节点评估信息，检查本轮是否完成。当所有评估批次完成后，系统调用交互总结智能体进行本轮总结。系统根据轮次计数决定继续或停止。如果未达到最大轮次，系统执行策略规划，构建上下文信息，调用大语言模型生成新策略，批量分发任务，进入下一轮迭代。如果达到最大轮次或满足停止条件，系统保存实验结果和可视化数据，发送任务完成消息给假设生成者。假设生成者综合所有活跃节点证据，生成最终科学报告，通过 WebSocket 推送到前端。

### 6.2 关键控制机制

系统通过多个计数器控制执行流程。轮次计数器记录当前轮次，最大轮次限制为7轮。待处理任务计数器记录工具执行任务数，待处理评估批次计数器记录评估批次数。系统通过去重机制避免重复处理相同证据，基于节点ID进行去重，重复节点更新命中次数但不重复评估。系统维护历史策略记录，用于避免重复策略，提高策略新颖性。

## 七、关键技术特性

### 7.1 多智能体协作

系统基于 AutoGen Core 框架实现多智能体协作，使用基于主题的消息路由机制，支持异步并发执行。各智能体通过标准化的消息协议进行通信，实现松耦合的协作模式。

### 7.2 动态策略规划

系统采用大语言模型驱动的策略生成机制，基于评估反馈动态调整检索策略，实现探索与利用的平衡。策略生成过程考虑历史策略、当前图谱状态、评估结果等多方面因素，确保策略的相关性和新颖性。

### 7.3 多模态检索支持

系统支持多种检索方式。文本语义检索基于向量相似度，支持自然语言查询。元数据精确匹配基于规则过滤，支持快速定位。图片多模态检索基于 CLIP 模型，支持图片相似度匹配。证据相似度检索基于嵌入向量，支持发现关联证据。

### 7.4 实时可视化支持

系统通过 WebSocket 实现实时数据推送，支持图谱结构动态更新、词云数据展示、质量评估可视化等。前端可以实时查看系统执行状态和结果。

### 7.5 完整数据记录

系统完整记录每轮迭代的原始数据，包括检索策略、检索结果、评估结果、统计信息等，形成完整的实验数据链，支持后续分析和论文撰写。

## 八、系统优化与设计亮点

### 8.1 性能优化

系统采用多种性能优化策略。并发执行机制支持多策略同时检索，显著提高执行效率。批处理评估机制将证据分批评估，避免大语言模型上下文过长导致的性能问题。本地缓存机制使用 JSON 文件缓存，支持快速过滤查询。自适应阈值机制在语义检索中动态调整分数阈值，确保返回足够数量的结果。

### 8.2 鲁棒性设计

系统具备完善的错误处理和恢复机制。异常捕获机制确保单个任务失败不影响整体流程。数据验证机制使用 Pydantic 进行类型安全验证。去重机制通过多层次策略避免重复处理。连接管理机制自动清理失效的 WebSocket 连接。

### 8.3 可扩展性

系统设计支持灵活扩展。工具插件化机制通过工具映射表轻松添加新工具。集合管理机制支持多个向量集合，可灵活切换。消息协议标准化确保新功能易于集成。

### 8.4 可观测性

系统提供完善的日志和调试支持。详细日志记录所有关键操作，包括策略生成、工具执行、评估结果等。实验数据完整保存每轮迭代的原始数据。调试日志记录关键执行路径，便于问题定位。

## 九、总结

ToRAGLENS 后端系统通过多智能体协作架构，实现了智能化的检索策略规划、多层次的检索能力、结构化的证据组织、实时的交互反馈、完整的数据记录等核心功能。系统设计充分考虑了性能、鲁棒性和可扩展性，通过并发执行、批处理、缓存、自适应阈值等优化策略提升性能，通过错误处理、数据验证、去重、连接管理等机制保障鲁棒性，通过插件化、标准化、模块化设计支持扩展。系统为科学知识发现提供了强大的技术支撑，支持从问题提出到报告生成的完整流程，为科研工作者提供了高效的知识发现工具。
